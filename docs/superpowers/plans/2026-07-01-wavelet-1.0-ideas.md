# Wavelet 1.0: Direction Ideas (post-beta)

Date: 2026-07-01
Status: IDEAS, not commitments. Nothing here is scheduled; Standing Rule 1 applies
(no feature without a consumer). The beta (`2026-07-01-wavelet-beta-remaining.md`)
ships first; this doc is the shape of the release after it.
Author priorities driving this doc: (a) arithmetic speed by growing the numeric
tower, not by leaning on bignum alone; (b) the language feeling homey: ergonomics,
speed, and errors that are easy to grok and "just work"; (c) the IaC layer usable
standalone or as components inside a broader program, with proofs of everything as
the center, never as an ecosystem you are locked into.

## Framing: what 1.0 means

Beta proves the thesis on one artifact. 1.0 means a stranger can pick Wavelet up,
be productive in an afternoon, ship something real, and never feel punished for
choosing it. Concretely: predictable performance, world-class errors, and an IaC
layer that composes with what they already have. The kernel stays sacred
throughout; everything below is codegen, elaborator, stdlib, or tooling strata.
Pre-1.0 the internal Go surfaces are free to change to fix bad design (recorded
project policy); 1.0 is when that freedom ends, so spend it here.

## Pillar 1: arithmetic that is fast because the tower is right

Current state: the core numeral is `NatLit{*big.Int}` (compressed, kernel-accel
ι on literals); native C/LLVM runtimes carry an arbitrary-precision bigint; the
int64 fast-lane was parked as "no consumer." Wavelet performance is now the
consumer. The principle: representation is a codegen concern, the kernel never
changes, and every fast lane is bound to the proven reference the way OpenBLAS
was (tolerance/equality contract, blame on mismatch).

- **Fixnum fast lane with overflow promotion.** Every backend represents a Nat/Int
  below 2^63 as a machine word and promotes to bignum on overflow (the Python-int
  / Lisp-fixnum discipline). The accel ι already fires only on literals, so open-
  term reasoning is untouched. Acceptance: the ch-level arithmetic benchmarks stop
  allocating; `leb` on a bignum in compiled code no longer OOMs (the known
  eager-elim trap); byte-identical conformance holds because values, not
  representations, are compared.
- **Machine word types as proven-bounded types.** `Word8/16/32/64` as stdlib types
  carrying their bound as a proposition (a Sigma over Nat with a proven bound, or
  a dedicated builtin group if a consumer demands it). Bit-level ops (and, or,
  xor, shifts) with proven algebraic laws; the Bin byte type already landed, so
  words complete the systems-programming floor.
- **The tower grows where use demands.** Nat -> Whole -> Int -> Rat exists; f64
  Float exists with contract-guarded BLAS. Candidates in priority order, each
  gated by a real consumer: (1) fixnum lanes above, (2) Word types, (3) exact
  Decimal (money; the IaC cost-estimation story wants it), (4) arbitrary-precision
  rationals fast path, (5) interval/fixed-point only if a verified-numerics
  consumer appears.
- **Eliminator-compilation honesty.** Case arms evaluate eagerly (case = eliminator
  application); the k^depth blowup and the eager NatElim loop are known traps.
  1.0 wants either a documented discipline (precompute predicates, single
  decision, recursive ih in exactly one arm) enforced by a lint, or a codegen
  demand-analysis that makes the trap impossible. The lint is the cheap first
  step.

## Pillar 2: homey ergonomics, speed, and errors

The bar: an error message teaches; a slow thing is slow for a reason you can see;
defaults are the thing you would have picked. Elm-quality errors and Go-quality
tooling, with proofs.

- **Diagnostics to full coverage.** The Diagnostic type + 11 families + parse
  carets landed. 1.0: every user-reachable failure (elaboration, instance search,
  QTT usage, ledger gate, infra apply, simulate divergence) speaks Diagnostic,
  with a one-line "what", the caret, and a "how to fix" tuned per family.
  Terraform/compose failures surfaced through the same shape, so the IaC boundary
  does not suddenly feel like a different, hostile tool.
- **The REPL is the front door.** Parity is policy (a feature is not done until it
  works in `repl` with a test). 1.0 adds: startup under 100ms with the prelude
  (snapshot or lazy-load the builtin groups), `:type`/`:hash`/`:ledger` inspection
  verbs, and error recovery that never loses the session.
- **LSP.** Hover types, go-to-definition via content hash, inline diagnostics,
  proof-status gutter (proven/guarded/assumed from the ledger). The single
  highest-leverage adoption feature; content addressing makes it unusually easy
  (identity is a hash, not a fragile name).
- **Compile and emitted-code speed as gates.** A small benchmark corpus (parse,
  check, emit, run) tracked per release the way conformance is. ARC replacing
  mark-sweep on C/LLVM (beta item 6e) folds in here. Nothing lands that
  regresses the corpus without a recorded reason.
- **"Just works" setup.** `bin/setup.sh` + `.tool-versions` exist; 1.0 wants one
  command from clean machine to green suite, and a `wvl doctor` that names any
  missing toolchain and what it gates (the DEVELOPMENT.md table, executable).
- **Surface polish debts.** Contextual keywords audited (the known collision
  trap); numeric literal ergonomics already good (decimal literals, Num
  overloading, negation promotion); string interpolation landed. Remaining: the
  honest survey of what a newcomer types first and where they stub their toe;
  run it as a usability pass with fresh eyes, fix the top ten.

## Pillar 3: IaC as a part, never a prison

The anti-Winglang stance, stated positively: Wavelet emits and consumes open
artifacts, and every layer is detachable. A team should be able to adopt exactly
one slice (just the Terraform emitter, just the simulator, just the proven CRDT
library) inside their existing program and pipeline, and expand later, or not.

- **Standalone mode (already mostly true, make it a product surface).**
  `wvl deploy` in, standard artifacts out: fmt-clean OpenTofu/Terraform HCL,
  Compose specs, CALM JSON, escript/WASM workloads. Any existing pipeline
  (Atlantis, Spacelift, plain CI) applies them; Wavelet does not need to be the
  runner. The assurance ledger travels alongside as a machine-readable artifact,
  so the proofs remain inspectable downstream of us.
- **Embedded mode (the growth edge).** `rune build --kind library` exists Go-only.
  1.0 extends library emission to the other source backends (JS/TS package, Python
  wheel, JVM jar, Rust crate, BEAM app), so a verified component (a CRDT, a
  policy check, a protocol) drops into a host program as an ordinary dependency
  with a typed host ABI. The infra layer itself is importable the same way: build
  a resource graph programmatically from host code, get HCL + ledger back.
- **Ingest, not just emit.** CALM round-trip already validates our own emission.
  The 1.0 idea: ingest an existing CALM doc (or a Terraform plan JSON) and report
  which claims Wavelet can prove, which it can guard, and which remain assumed.
  Adoption path for brownfield: start by getting an honest ledger over what you
  already have, then migrate pieces into the proven tier.
- **No private formats.** Manifest is plain text; identities are content hashes;
  emitted code is dependency-free source in the target language. The exit cost
  from Wavelet is: keep the emitted artifacts, lose the ability to regenerate and
  re-prove. That is the whole lock-in, and it is the right amount: the proofs are
  the reason to stay.

## Pillar 4: proofs at the center (the invariant that makes 1.0 coherent)

Everything above is ordinary language-tooling work; this is the part only Wavelet
can promise. Every 1.0 feature must preserve:

- **The tier is visible everywhere.** proven / guarded / assumed / postulate on
  every control, every foreign claim, every fast lane. The LSP gutter, the ledger
  artifact, the CI gate, and the docs all read the same source of truth.
- **Fast lanes are bound to references.** A fixnum lane, an OpenBLAS call, a
  numpy bridge: each is either proven equivalent or contract-guarded with blame.
  No silent trust.
- **The kernel does not grow for product reasons.** Hash format stays 0x06 unless
  a genuinely new core constructor is required by mathematics, not marketing.
- **Byte-identical conformance is non-negotiable.** New backends and new fast
  lanes join the gate or they do not ship.

## What 1.0 is explicitly not

- Not multi-cloud LIVE orchestration with drift detection (emit + apply + ledger
  is the product; fleet management is somebody else's).
- Not an SMT-backed verifier, not a tactic framework (NON-GOALS.md holds).
- Not a package registry or codebase manager (the ucm-scale project stays out;
  the flat name -> hash map plus ordinary language package managers suffice).
- Not source-compatible forever: 1.0 is where surface stability begins, not
  earlier.

## Sequencing sketch (dependency, not calendar)

1. Beta ships (the other doc).
2. Fixnum fast lane + benchmark corpus (pillar 1 core, unlocks honest speed
   claims for everything after).
3. Diagnostics full coverage + `wvl doctor` + REPL polish (pillar 2 floor).
4. LSP (pillar 2 ceiling; biggest adoption lever).
5. Library emission beyond Go + programmatic infra graphs (pillar 3 embedded).
6. CALM/Terraform ingest with honest-ledger report (pillar 3 brownfield).
7. Word types / Decimal as consumers appear (pillar 1 tail).
