# scribe Rune port L3: pure exact rasterizer (ch563)

> Executor plan. Spec: scribe repo docs/superpowers/specs/2026-07-02-rune-port-spec.md.
> ch561/ch562 lessons all apply (one ih, thunked arms, point-free fuel towers,
> gcd-reduced Q, bytewise word compares).

**Goal:** `listings/ch563_scribe_raster.rune`: the font-rs signed-area
rasterizer over exact rationals. Coverage is an exact rational; alpha is
`min(255, floor(cov * 256))` by integer arithmetic, so the Go engine's
"half coverage is exactly 128" TEST becomes an arithmetic FACT. Plus the
first Go-vs-rune cross-language lock: a 32x32 continuous roundrect fill,
rendered by Go scribe into a committed fixture, must match the rune
rasterizer's output alpha for alpha.

## Design pinned up front

- Chapter redeclares the ch562 geometry verbatim (corpus convention): Qn
  (gcd-reduced), Pt, corner tables, builders, worklist flattener. New here:
  qsnap (round to the 1/256 grid, half away from zero, exactly Go's
  math.Round(v*256)/256), the accumulation buffer, the edge walk, and the
  alpha pass.
- **Buffer**: flat `List Qn` of (w+1)*h cells, functional single-cell update
  by a fuel walk. Slow is fine: the spec scopes L3 to small canvases; the
  fast path is L4's host op.
- **Edge walk**: the exact structure of Go raster.line() in Q: skip
  horizontal, orient upward, dxdy exact, y-clip, per-scanline dy, single-cell
  vs multi-cell deposits with the a0/am/a1 formulas, x clamped to [0, w],
  guard column w. Everything is a fold; ONE ih per loop; branch bodies
  thunked.
- **Alpha pass**: per row, prefix-sum fold; nonzero rule cov = min(1, |acc|);
  alpha = min(255, floor(cov*256)) via fdiv on exact numerator/denominator.
- **Pins** (the Go raster tests, exact): full-pixel rect pattern on 4x4;
  half-coverage row = 128 on 4x3; vertical half = 128/128; diagonal triangle
  = 128; quarter = 64; full-canvas clip rect = all 255 on 2x2.
- **The lock instance**: canvas 32x32, RoundRect(4,4,24,24) radius 6
  continuous, flatten tol 1/20 (Go FlattenTol 0.05), vertices snapped 1/256,
  nonzero. Go fixture generated by a scribe-repo program writing 1024 alpha
  lines; committed under harness/testdata/scribe_lock/. A new harness gate
  emits ch563's `main` (prints the 1024 alphas) and diffs. HONEST RISK,
  handled empirically: Go flattens in float64, rune exactly; if subdivision
  or snap boundaries diverge the masks may differ, and the finding (where
  and why) is part of L3's deliverable. The in-listing `answer` (alpha sum)
  pins kernel-vs-backend agreement regardless.
